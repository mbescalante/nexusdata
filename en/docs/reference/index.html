Referencia API

Documentación completa de todos los endpoints, métodos y modelos disponibles en NexusData API.

Endpoints REST​

Usuarios​

Obtener Usuario​

GET /api/users/{id}

Parámetros:​

  • id (obligatorio): Identificador único del usuario Respuesta exitosa:

    {
    "id": "123",
    "name": "John Doe",
    "email": "john@example.com",
    "createdAt": "2023-01-15T14:30:00Z"
    }

    Listar Usuarios​

    GET /api/users?limit=10&offset=0

Parámetros de consulta:​

  • limit (opcional): Número máximo de resultados (predeterminado: 10)
  • offset (opcional): Número de resultados a omitir (predeterminado: 0)
  • sort (opcional): Campo por el cual ordenar (ejemplo: name )
  • order (opcional): Dirección de ordenamiento ( asc o desc )

Crear Usuario:​

POST /api/users

Cuerpo de la solicitud:​

  {
  "name": "John Doe",
  "email": "john@example.com",
  "password": "securePassword123"
  }

Respuesta exitosa:​

  {
  "id": "123",
  "name": "John Doe",
  "email": "john@example.com",
  "createdAt": "2023-06-15T14:30:00Z"
  }

Actualizar Usuario:​

  PUT /api/users/{id}

Parámetros:​

  • id (obligatorio): Identificador único del usuario Cuerpo de la solicitud:
  {
  "name": "Jane Doe",
  "email": "jane@example.com"
}

Eliminar Usuario

DELETE /api/users/{id}

Parámetros:

  • id (obligatorio): Identificador único del usuario

Productos Obtener Producto​

GET /api/products/{id}

Parámetros:

  • id (obligatorio): Identificador único del producto

Respuesta exitosa:

{
"id": "456",
"name": "Smartphone XYZ",
"price": 599.99,
"description": "El último modelo con características avanzadas",
"stock": 42,
"createdAt": "2023-05-10T09:15:00Z"
}

Listar Productos

GET /api/products?category=electronics&limit=20

Parámetros de consulta:

  • category (opcional): Filtrar por categoría
  • minPrice (opcional): Precio mínimo
  • maxPrice (opcional): Precio máximo
  • inStock (opcional): Filtrar productos en stock ( true o false )
  • limit (opcional): Número máximo de resultados
  • offset (opcional): Número de resultados a omitir Crear Producto
POST /api/products

Cuerpo de la solicitud:​

{
  "name": "Smartphone XYZ",
  "price": 599.99,
  "description": "El último modelo con características avanzadas",
  "categoryId": "electronics",
  "stock": 42
}

Actualizar Producto​

PUT /api/products/{id}

Parámetros:​

  • id (obligatorio): Identificador único del producto

Cuerpo de la solicitud:​

{
  "price": 549.99,
  "stock": 38
}

Eliminar Producto​

DELETE /api/products/{id}

Parámetros:​

  • id (obligatorio): Identificador único del producto

Endpoints GraphQL​

Usuarios Obtener Usuario​

query GetUser($id: ID!) {
  user(id: $id) {
    id
    name
    email
    createdAt
    roles {
      id
      name
    }
  }
}

Variables:​

{
  "id": "123"
}

Listar Usuarios​

query ListUsers($limit: Int, $offset: Int, $filter: UserFilterInput) {
  users(limit: $limit, offset: $offset, filter: $filter) {
    items {
      id
      name
      email
      createdAt
    }
    total
    hasMore
  }
}

Variables:​

{
  "limit": 10,
  "offset": 0,
  "filter": {
    "nameContains": "John"
  }
}

Crear Usuario​

mutation CreateUser($input: CreateUserInput!) {

    createdAt

Productos Obtener Producto​

query GetProduct($id: ID!) {
  product(id: $id) {
    id
    name
    price
    description
    stock
    category {
      id
      name
    }
    createdAt
  }
}

Variables:​

{
  "id": "456"
}

Listar Productos​

query ListProducts($filter: ProductFilterInput, $limit: Int, $offset: Int) {
  products(filter: $filter, limit: $limit, offset: $offset) {
    items {
      id
      name
      price
      stock
      category {
        name
      }
    }
    total
    hasMore
  }
}

Variables:​

{
  "filter": {
    "categoryId": "electronics",
    "minPrice": 100,
    "inStock": true
  },
  "limit": 20,
  "offset": 0
}

Códigos de Estado HTTP Código Descripción 200​

CódigoDescripciónSignificado
200OKLa solicitud se ha completado correctamente.
201CreatedEl recurso se ha creado correctamente.
400Bad RequestLa solicitud contiene datos inválidos.
401UnauthorizedAutenticación requerida.
403ForbiddenNo tiene permisos para acceder al recurso.
404Not FoundEl recurso solicitado no existe.
422Unprocessable EntityError de validación.
500Internal Server ErrorError del servidor.
